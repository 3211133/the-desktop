# 設定ファイルの検証と永続化タイミング

## 現状
- `load_config()` は JSON 読み込み失敗時にデフォルトへフォールバックする。
- `window.x/y/w/h` は `int(...)` へ直接変換している。
- `topmost` は `bool(raw.get("topmost", True))` で解釈している。
- 設定保存は `_quit()` 経由でのみ呼ばれ、トレイ格納や最前面トグル直後には保存されない。

## 課題
1. 型の壊れた設定に弱い。
   - `window.w` が `"abc"` のような値だと `ValueError` で `load_config()` 全体が失敗する可能性がある。
   - `topmost` が文字列 `"false"` の場合、Python の `bool("false")` は True になる。
2. 画面外・極端なサイズへの復元リスクがある。
   - マルチモニター構成変更後に、保存済み座標が画面外になる可能性がある。
   - 幅・高さが 0 や負数、極端に小さい値でもそのまま使われる。
3. 永続化タイミングが限定的。
   - × ボタンは「終了せずトレイに格納」なので、ユーザーは終了操作をあまり使わない可能性がある。
   - その場合、位置・サイズや最前面設定が保存されないままになる。

## 対応案
- 設定値ごとに安全な parser を用意し、不正値は項目単位でデフォルトに戻す。
- `topmost` は bool 型のみ受け付ける、または文字列を明示的に `true` / `false` として parse する。
- geometry は最小サイズを clamp し、起動時に現在のスクリーン範囲へ収める。
- `moveEvent` / `resizeEvent` / 最前面トグル時に debounce 付きで保存する、またはトレイ格納時にも保存する。

## 優先度
中。即座にクラッシュへつながるケースは限定的だが、常駐アプリでは設定破損・画面外復元の体験悪化が目立ちやすい。
